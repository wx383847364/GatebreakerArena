# Gatebreaker Arena 英雄芯片共鸣路径设计 v0.2

> **版本**: v0.2 | **日期**: 2026-07-07 | **作者**: GameDesigner
> **状态**: 方案阶段
> **依赖**: 英雄系统设计 v0.1, 局外养成与随机触发 v0.1, 局内变数系统设计 v0.1, 热更新边界规范 v1
> **变更类型**: 设计哲学重构 — 英雄基础削弱 × 芯片带来质性改变 × 多路径方向

---

## 0. 设计哲学重构

### 0.1 为什么需要重构

v0.1 的英雄系统存在一个核心矛盾：

| 问题 | 具体表现 |
|---|---|
| **英雄太完整** | 冰雪女王自带 +15 寒气/次、1.5s 冰冻、暴风雪技能——没有芯片她就已经是一套完整武器 |
| **芯片太弱** | 芯片只是 "+20% 球速" 这类数值微调，感觉像是"锦上添花"而非"质的飞跃" |
| **玩法单一** | 每个英雄只有一种玩法——冰雪女王永远是"累积寒气→冰冻→暴风雪收割" |
| **养成无力感** | "我攒了两周解锁的芯片，只让我的英雄快了 20%？这没有仪式感。" |

### 0.2 新设计哲学：英雄是画布，芯片是颜料

| 新铁律 | 含义 | 违反后果 |
|---|---|---|
| **英雄基础有意弱化** | 英雄的被动/主动只提供"骨架"和"主题"，不是完整武器 | 英雄选了就定胜负，芯片无意义 |
| **芯片带来质性改变** | 芯片不只调数值，而是改变英雄技能的行为模式 | 芯片只是"+20%"的数字游戏，养成无仪式感 |
| **每英雄多路径方向** | 同一英雄有 2-3 条芯片共鸣路径，每条方向玩法完全不同 | 英雄选了就定了唯一打法，build 无策略深度 |
| **路径间互斥激励** | 深入一条路径获得的质性奖励比浅尝三条更强 | 玩家随便凑芯片，路径无身份感 |

**玩家体验目标**：

> "我选了冰雪女王，这局觉醒了攻+守芯片，走极寒之路——冰冻变成了全场冰冻，暴风雪变成了冰墙封锁。下一局同样的英雄，但觉醒了乱类芯片，走冰暴之路——寒气不再冰冻挡板，而是让所有球减速。**同一个英雄，两种完全不同的体验。**"

### 0.3 层级关系更新

```
英雄 (骨架层)      ← 你选谁，决定主题和基础骨架（有意弱化）
  × 芯片共鸣路径 (方向层) ← 觉醒的芯片匹配哪条路径，决定玩法方向
    × 路径里程碑 (质性层) ← 匹配的路径芯片越多，质性改变越大
      × 地图变量 (环境层) ← 场地结构变化
        × 局内道具 (即时层) ← 临时 buff/debuff
```

五层独立运作，英雄+路径里程碑产生**乘法质性叠加**。

## 1. 英雄基础数据削弱方案

### 1.1 削弱原则

英雄的基础被动/主动只提供**"主题辨识度"**，而非**"竞技武器"**。玩家应该在 10 秒内能感知到"这个英雄有什么特色"，但不应在没有芯片时感到"我已经很强了"。

| 属性 | v0.1 旧值 | v0.2 新值 | 削弱幅度 | 设计意图 |
|---|---|---|---|---|
| 冰雪女王 寒气累积 | +15/次 | +5/次 | -67% | 需要 ~20 次挡板反弹才能触发冰冻，单纯靠被动几乎不可能 |
| 冰雪女王 冰冻时长 | 1.5s | 0.6s | -60% | 0.6s 只够让对手"稍微愣一下"，不是致命控制 |
| 冰雪女王 暴风雪翻倍 | ×2 | ×1.5 | -25% | 翻倍变为 +7.5/次，仍然很慢 |
| 烈焰领主 热量累积 | +12/次 | +4/次 | -67% | 同冰雪女王逻辑 |
| 烈焰领主 减速幅度 | -50% | -20% | -60% | 只是轻微减速，不是致命减益 |
| 雷霆审判者 偏转角度 | 5°~8° | 2°~3° | -63% | 微乎其微的偏转，可感知但几乎无战术意义 |
| 重力使者 引力加速度 | 0.3/s² | 0.1/s² | -67% | 球路微微弯曲，不明显 |
| 时之织者 减速幅度 | -15% | -8% | -47% | 只是"球稍微慢了一点" |
| 荆棘守护者 挡板生长 | +8%/次 | +3%/次 | -63% | 3 层才 +9%，几乎无影响 |
| 荆棘守护者 弹药扣除 | 1 发 | 0 发 → 0.5 发概率 | -50% | 概率性扣半发，不是确定性惩罚 |
| 机械工程师 屏障间隔 | 15s | 20s | -33% | 一局只有 3 面屏障 |
| 暗影行者 不透明度 | 30% | 50% | +67% | 更容易看到，隐蔽性大幅降低 |
| 辉光圣骑 蓄力层数 | 5 层 | 8 层 | +60% | 需要更多防守才能爆发 |
| 辉光圣骑 爆发增益 | +100% | +30% | -70% | 爆发只是"稍微快一点" |
| 量子术士 隧穿概率 | 15% | 5% | -67% | 基本不触发 |

### 1.2 代价同步调整

代价是英雄的**永久约束**，不能被芯片完全消除。但代价幅度也需要下调，因为英雄基础已经弱了——代价和被动应该保持比例关系：

| 英雄 | v0.1 代价 | v0.2 代价 | 调整说明 |
|---|---|---|---|
| 冰雪女王 | 球速 -10%/-8% | 球速 -5% | 与削弱后的被动匹配 |
| 烈焰领主 | 弹药 -1 | 弹药上限不变，发球冷却 +10% | 弹药-1 在弱被动下太伤 |
| 雷霆审判者 | 挡板 -10% | 挡板 -5% | 微弱偏转配微弱代价 |
| 重力使者 | 发球速 -15% | 发球速 -8% | 微弱引力配微弱代价 |
| 时之织者 | 球速上限 -10% | 球速上限 -5% | 微弱减速配微弱代价 |
| 荆棘守护者 | 发球速 -20% | 发球速 -10% | 微弱反伤配微弱代价 |
| 机械工程师 | 挡板速 -10% | 挡板速 -5% | 微弱屏障配微弱代价 |
| 暗影行者 | 挡板 -15% | 挡板 -8% | 弱隐蔽配弱代价 |
| 辉光圣骑 | 挡板移速 -12% | 挡板移速 -6% | 弱蓄力配弱代价 |
| 量子术士 | 8%球失归属 | 3%球失归属 | 弱隧穿配弱代价 |

### 1.3 主动技能同步削弱

| 英雄 | v0.1 主动 | v0.2 主动 | 调整说明 |
|---|---|---|---|
| 冰雪女王 | 暴风雪: 翻倍累积 5s | 暴风雪: 累积 ×1.5，持续 4s | 弱化但仍可感知 |
| 烈焰领主 | 地狱火球: +50%速/+50%体积/穿1次墙 | 地狱火球: +20%速/+20%体积 | 不穿墙，只有微弱增强 |
| 雷霆审判者 | 雷霆审判: +50%速/追踪敌门4s | 雷霆审判: +20%速/2°追踪3s | 微弱加速+几乎无追踪 |
| 重力使者 | 奇点: 强引力井4s | 奇点: 中等引力井3s | 吸力减弱 |
| 时之织者 | 时间静止: 冻敌球1.5s | 时间静止: 冻敌球0.5s | 极短冻结 |
| 荆棘守护者 | 荆棘装甲: +80%速反弹6s | 荆棘装甲: +30%速反弹4s | 微弱反弹增强 |
| 机械工程师 | 据点: 门前墙5s | 据点: 门前墙3s | 缩短防护时间 |
| 暗影行者 | 相位球: 穿墙+全程暗影 | 相位球: 穿1次墙+60%不透明度 | 不全程暗影 |
| 辉光圣骑 | 圣盾: 门免疫3s | 圣盾: 门免疫1.5s | 极短免疫 |
| 量子术士 | 量子分裂: 球翻倍 | 量子分裂: 50%概率分裂(球速×0.7) | 不确定性分裂 |

**核心感觉**: 没有芯片的英雄，被动像是"气味"——你能闻到，但不构成威胁。主动像是"工具"——你能用，但不改变局势。**芯片才是武器。**

## 2. 芯片共鸣路径框架

### 2.1 路径定义结构

每个英雄定义 2-3 条**芯片共鸣路径**（Resonance Path）。每条路径包含：

```csharp
public class HeroResonancePath
{
    public string PathId;              // 路径唯一标识 (如 "PATH_FROST_DEEP_FREEZE")
    public string DisplayName;         // 路径名 (如 "极寒之路")
    public string Description;         // 路径主题描述
    public ChipCategory[] PathCategories;  // 该路径关联的芯片类别组合
    public ResonanceMilestone[] Milestones; // 里程碑：质性升级
}
```

### 2.2 里程碑机制

**路径里程碑** 是关键设计——当玩家觉醒的芯片中有足够多匹配某条路径类别的芯片时，解锁对应的质性升级：

| 里程碑等级 | 触发条件 | 升级类型 |
|---|---|---|
| **M1 觉醒** | 1 张路径芯片觉醒 | 数值强化（累积速度×2，时长+50%等） |
| **M2 质变** | 2 张路径芯片觉醒 | **质性改变**——技能行为模式改变 |
| **M3 极化** | 3 张路径芯片觉醒 | **终极质变**——新增子机制或完全重构技能 |

**重要规则**：
- 里程碑只认**芯片类别**，不认具体芯片ID——这保证了路径门槛可控且与牌组策略一致
- 同一英雄可能同时触发多条路径的 M1，但 M2/M3 只在**深入一条路径**时才有意义
- 路径间有**互斥激励**：3 张同类芯片深入一条路径获得的 M3 质变 > 3 张各不同类浅尝三条路径的 3 个 M1 数值增强

### 2.3 路径类别组合设计

芯片类别参考局外养成系统的四分类（攻 Strike/守 Guard/速 Flow/乱 Chaos）。每条路径由 2 个类别组合定义：

| 组合 | 主题 | 质性方向 |
|---|---|---|
| 攻+守 | **极致化** | 把英雄原有机制推向极致（寒气→全场冰冻） |
| 攻+乱 | **变异化** | 改变英雄原有机制的行为方式（寒气→减速而非冰冻） |
| 守+速 | **反转化** | 把英雄的代价反转为优势（寒气→己方加速） |
| 速+乱 | **解构化** | 拆解英雄机制为更碎片化的效果（冰冻→随机冰冻） |
| 攻+速 | **爆发化** | 原有机制频率大幅提升（寒气累积×3+衰减停止） |
| 守+乱 | **场控化** | 原有机制从单体变场地（冰冻→场地减速区） |

## 3. 十英雄共鸣路径完整设计

### 3.1 冰雪女王 (Frost Queen)

**基础骨架（有意弱化）**：球带寒气 +5/次，对手累积到 100 时冰冻 0.6s，暴风雪 ×1.5 持续 4s。

#### 路径 A：极寒之路 (攻+守)

**主题**："把冰冻推向极致——让对手完全冻结，让暴风雪封锁场地。"

| 里程碑 | 触发条件 | 质性升级 |
|---|---|---|
| **M1 觉醒** | 1 张攻/守芯片 | 寒气累积 ×2（+10/次），冰冻时长 ×1.5（0.9s），暴风雪持续 ×1.25（5s） |
| **M2 质变** | 2 张攻/守芯片 | **冰冻扩展为全场冰冻**：冰冻触发时，对手挡板冰冻 + 对手**所有场上球冰冻** 1.5s（球原地停止）。暴风雪变为**冰墙封锁**：在场地上生成一道冰墙（长 2 单位，持续 4s），对手球反弹，己方球穿透 |
| **M3 极化** | 3 张攻/守芯片 | **冰冻触发时生成冰刺区**：冰冻结束后，对手挡板周围 3 单位内形成冰刺区，对手球经过时减速 30% 持续 3s。暴风雪变为**极寒风暴**：累积 ×3（+15/次），且冰墙变为**三道冰墙**围住对手半场 |

**玩家幻想**：
- M1: "我的冰冻终于有感觉了！"
- M2: "冰冻不止冻挡板，球也冻了！暴风雪变成了冰墙——对手被封住了！"
- M3: "对手即使从冰冻恢复，也要经过冰刺减速区——冰冻变成了持续控制链！"

#### 路径 B：冰暴之路 (攻+乱)

**主题**："寒气不再冰冻对手，而是让球减速——从控制型变成压迫型。"

| 里程碑 | 触发条件 | 质性升级 |
|---|---|---|
| **M1 觉醒** | 1 张攻/乱芯片 | 寒气累积 ×2（+10/次），触发阈值降至 80 |
| **M2 质变** | 2 张攻/乱芯片 | **冰冻变异为冰暴减速**：寒气达到阈值时不再冰冻挡板，而是**对手所有场上球减速 40% 持续 3s**。暴风雪变异为**冰弹分裂**：己方场上球分裂为 2 颗（速度 ×0.8），分裂球同样携带寒气 |
| **M3 极化** | 3 张攻/乱芯片 | **冰暴减速 + 冰面蔓延**：减速触发时，场地上生成冰面区域（半径 3 单位，持续 5s），任何球经过冰面都减速 20%。冰弹分裂变为**三分裂**（3 颗，速度 ×0.65），分裂球墙弹时 20% 概率再分裂 1 次 |

**玩家幻想**：
- M1: "累积更快了，可以更频繁触发。"
- M2: "冰冻变成了全场球减速！而且暴风雪让球分裂——场面爆炸！"
- M3: "减速不只是球的——场地本身变成了冰面！分裂球还能再分裂！"

#### 路径 C：冰晶之路 (守+速)

**主题**："寒气不再是惩罚对手——而是强化自己。冰冻变成冰镜，反弹攻击。"

| 里程碑 | 触发条件 | 质性升级 |
|---|---|---|
| **M1 觉醒** | 1 张守/速芯片 | 寒气衰减停止（对手不碰球也不会消退），冰冻时长 ×1.3 |
| **M2 质变** | 2 张守/速芯片 | **寒气反转为己方加速**：寒气累积不再是给对手施加寒气，而是**己方球碰对手挡板时自身加速 +15%/次**。冰冻变异为**冰镜**：对手挡板碰到冰雪女王球时，球以 +50% 速度反弹向最近敌方球门 |
| **M3 极化** | 3 张守/速芯片 | **冰镜变为双面冰镜**：反弹球同时触发一次**冰棱折射**——反弹球在飞行 3 单位后分裂出一颗镜像球（角度偏转 30°），两颗球都朝对手门飞。暴风雪变异为**冰晶装甲**：己方挡板获得冰晶护甲 5s，挡板碰球反弹 +80% 速且偏向敌门 |

**玩家幻想**：
- M1: "寒气不衰减了——对手迟早要被冻。"
- M2: "我的寒气不再惩罚对手，而是让我的球越打越快！冰冻变成了反弹冰镜！"
- M3: "冰镜反弹的球会分裂出镜像球——两球齐射！暴风雪变成了冰晶装甲——我的挡板也能反击！"

### 3.2 烈焰领主 (Pyro Lord)

**基础骨架（有意弱化）**：球带灼烧 +4/次，热量到 100 时对手减速 20% 持续 2.5s，地狱火球 +20%速/+20%体积。

#### 路径 A：炼狱之路 (攻+守)

**主题**："灼烧变真烧——从减速变成持续灼烧伤害链。"

| 里程碑 | 触发条件 | 质性升级 |
|---|---|---|
| **M1 觉醒** | 1 张攻/守芯片 | 累积 ×2（+8/次），减速幅度 ×1.5（-30%），灼烧时长 ×1.25（3.1s） |
| **M2 质变** | 2 张攻/守芯片 | **灼烧升级为炼狱灼烧**：减速结束后对手挡板继续**灼烧减速 -15% 持续 3s**（总计约 6s 减速链）。地狱火球变为**炼狱火球**：球碰墙时向周围 2 单位内的对手球释放**灼烧脉冲**（+20 热量），被脉冲影响的球碰到其他挡板时热量传导一次 |
| **M3 极化** | 3 张攻/守芯片 | **炼狱灼烧变为熔岩铺地**：灼烧触发时对手半场生成**熔岩区**（持续 4s），对手球经过熔岩区减速 25%。炼狱火球变为**陨石火球**：球碰挡板/墙时爆炸，爆炸范围内的对手挡板直接累积 +50 热量 |

#### 路径 B：涅槃之路 (攻+速)

**主题**："火焰不再惩罚对手——它燃烧自己，换取爆发力。"

| 里程碑 | 触发条件 | 质性升级 |
|---|---|---|

## 4. 共鸣路径触发规则

### 4.1 里程碑触发计算

```
每名玩家的路径里程碑触发:
  1. 统计当前觉醒的芯片中各类别数量
  2. 对每个英雄路径，检查 PathCategories 组合匹配度
  3. 如果某路径的类别组合中有 N 张芯片觉醒 → 触发该路径的 M(N) 里程碑
  4. 多路径可以同时触发 M1（浅尝）
  5. M2/M3 只在深入一条路径时触发

示例:
  冰雪女王觉醒了: STRIKE_POWER(攻), GUARD_LENGTH(守), FLOW_SPEED(速)
  
  极寒之路(攻+守): 匹配 2 张 → M2 质变触发！
  冰暴之路(攻+乱): 匹配 1 张(攻) → M1 觉醒触发
  冰晶之路(守+速): 匹配 2 张(守+速) → M2 质变触发！
  
  → 极寒之路 M2 + 冰暴之路 M1 + 冰晶之路 M2 同时生效
  → 玩家需要选择"走哪条路深入"作为牌组策略
```

### 4.2 里程碑选择 UI

共鸣波动时（t=20s/40s/55s），如果新觉醒的芯片使某路径升级到下一个里程碑，**暂停游戏 1s** 展示里程碑升级动画：

```
里程碑升级展示:
  → 路径名称 + 里程碑等级 + 质性升级名称
  → 英雄特效颜色变化（不同路径不同色调）
  → 技能描述变更弹出
  → 音效: 低频共振 + 路径专属音效
```

### 4.3 路径互斥激励

为鼓励深入一条路径而非浅尝多条，M2/M3 的质性升级强度远超 M1：

| 里程碑 | 数值增强 | 质性改变 | 玩家感知 |
|---|---|---|---|
| M1 (1张) | 累积 ×2, 时长 ×1.5 | 无 | "稍微强了一点" |
| M2 (2张) | +M1 数值 | **技能行为模式改变** | "玩法完全不同了！" |
| M3 (3张) | +M2 数值 | **新增子机制/完全重构** | "这已经不是原来的英雄了！" |

**3 条 M1（浅尝）≈ 数值 ×2 × 3 方面 ≈ 还是在玩同一个英雄**
**1 条 M3（深入）≈ 质性重构 + 数值 ×3 ≈ 完全不同的英雄**

### 4.4 共鸣波动与路径里程碑的关系

| 时间点 | 觉醒芯片 | 路径里程碑变化 |
|---|---|---|
| t=0s | 3 张初始觉醒 | 可能触发 1~3 个 M1，或 1 个 M2 |
| t=20s | +1 张波动觉醒 | 可能升级某路径 M1→M2 |
| t=40s | +1 张波动觉醒 | 可能升级 M2→M3 |
| t=55s | +1 张超载觉醒 | 最可能触发 M3 或新路径 M1 |

**设计意图**: 随着对局推进，路径越来越深入——开局是"种子"，中盘是"生长"，终盘是"绽放"。

### 5.3 里程碑触发逻辑

```csharp
// 在 ResonanceSurgeSystem 中新增
public void CheckMilestones(PlayerRuntimeState player, ChipDefinition[] activeChips)
{
    var heroDef = HeroCatalog.Get(player.SelectedHeroId);
    
    foreach (var path in heroDef.ResonancePaths)
    {
        // 统计当前觉醒芯片中匹配路径类别的数量
        int matchCount = activeChips.Count(c => 
            path.PathCategories.Contains(c.Category));
        
        // 确定当前应该触发的里程碑等级
        int targetLevel = 0;
        foreach (var milestone in path.Milestones)
        {
            if (matchCount >= milestone.RequiredPathChipCount)
                targetLevel = milestone.MilestoneLevel;
        }
        
        // 如果里程碑升级，触发质变动画和效果
        int currentLevel = player.PathMilestoneLevels.GetValueOrDefault(path.PathId, 0);
        if (targetLevel > currentLevel)
        {
            TriggerMilestoneUpgrade(player, path, targetLevel);
            player.PathMilestoneLevels[path.PathId] = targetLevel;
        }
    }
}
```

## 8. 牌组策略与路径的关系

### 8.1 牌组构建新维度

v0.1 的牌组策略是"我想什么风格的可能性出现"。v0.2 增加了新维度——"我想走哪个路径"。

| 策略类型 | 牌组配置 | 路径效果 |
|---|---|---|
| **单路径深入** | 6 张攻+守芯片 + 2 张其他 | 极大概率触发 M2/M3 → 质性改变 |
| **双路径均衡** | 4 张攻 + 4 张守 | 可能触发两条路径的 M1/M2 → 两种质变浅尝 |
| **三路径广撒** | 每类 2 张 | 三条路径各 M1 → 纯数值增强无质变 |
| **路径+安全** | 4 张攻+守 + 4 张速 | M2/M3 路径深入 + 速类兜底 |

**关键决策**: 玩家需要在牌组编辑阶段决定"我牺牲多样性换取质变深度"还是"我保留灵活性但只有数值增强"。

### 8.2 觉醒概率与路径触发概率

| 牌组类型 | 3 张觉醒中匹配路径芯片的概率 | M2 触发概率 |
|---|---|---|
| 单路径深入(6攻+守/2其他) | 平均 1.5~2 张匹配 | ~40% |
| 双路径均衡(4攻/4守) | 平均 1~2 张匹配 | ~20%~30% |
| 均匀分布(每类2张) | 平均 0.75 张匹配 | ~10% |

共鸣波动（t=20s/40s/55s）会逐渐增加匹配芯片数量 → **中后期路径更深**。

### 8.3 镜像模式下的路径

排位模式使用镜像觉醒（双方共享芯片池），但**英雄仍各自选择**：

```
镜像模式下:
  1. 双方各自选择英雄 (可能不同)
  2. 双方各自组建牌组
  3. 合并牌组为统一池
  4. 从统一池抽取 3 张 → 双方觉醒相同芯片
  5. 但双方英雄不同 → 芯片匹配的路径不同 → 质性改变不同
  
  示例:
  玩家A选冰雪女王，玩家B选烈焰领主
  共同觉醒: STRIKE_POWER(攻), GUARD_LENGTH(守), CHAOS_SPIN(乱)
  
  玩家A:
    极寒之路(攻+守): 2张 → M2 质变！冰冻→全场冰冻
    冰暴之路(攻+乱): 2张 → M2 质变！冰冻→冰暴减速
    
  玩家B:
    炼狱之路(攻+守): 2张 → M2 质变！灼烧→炼狱灼烧链
    焦土之路(守+乱): 2张 → M2 质变！灼烧→焦土区
    
  → 同样的芯片，不同的英雄 → 完全不同的质性体验
```

**这正是设计的妙处**：镜像保证了公平（同样的芯片），英雄差异保证了多样性（不同的质性改变）。

## 11. 风险与缓解

### 11.1 新增风险

| 风险 | 严重度 | 缓解 |
|---|---|---|
| 90 个质性升级节点的实现工作量过大 | 高 | 分批实现：先 M1/M2，M3 远期；质性效果用枚举+参数化 |
| 路径互斥激励不够——玩家仍然浅尝 | 中 | M2 质性效果刻意设计得比 3 个 M1 数值增强更"爽" |
| 质性改变破坏核心玩法直觉 | 中 | 质性改变必须"可理解、可反制"——每个质变都有反制方式 |
| 英雄弱化让新手选英雄后感到无力 | 中 | 新手默认搭配免费芯片（已有3张基础芯片），弱化英雄+基础芯片 ≈ v0.1的英雄 |
| 镜像模式下不同英雄路径强度差异 | 中 | 质性改变按英雄统一设计框架，各路径 M2 约为等强度质变 |

### 11.2 确定性风险（继承并扩展）

| 风险 | 严重度 | 缓解 |
|---|---|---|
| 里程碑触发多端不同步 | 高 | 里程碑等级纳入 checksum |
| 质性改变效果脚本多端不一致 | 高 | 质性效果用确定性参数化实现，不用动态脚本 |
| 共鸣波动触发里程碑展示动画与 Lockstep 冲突 | 中 | 展示动画是纯 UI 层，不影响物理 tick |
