## 0. 文档定位

本文档定义 Gatebreaker Arena 的**改装机系统**——局外养成的第一支柱。

改装机是接入者在局外选择并带入对战的**机器配置**，每台改装机拥有：
- 一个**被动模块**（整局持续生效，定义改装机风格）
- 一个**主动模块**（玩家手动触发，有冷却）
- 一个**代价/限制**（确保竞技平衡）

改装机是模块系统的**基底**：改装机定义"你用什么机器"，模块定义"这局你额外激活了什么"。两者乘法叠加。

### 与现有系统的层级关系

```
改装机 (身份层)     ← 你选哪台，决定被动+主动+代价
  × 模块激活 (随机层)  ← 每局随机激活的修饰器
    × 地图变量 (环境层)  ← 场地结构变化
      × 局内道具 (即时层) ← 临时buff/debuff
```

四层独立运作，乘法叠加，最终形成一局对战的有效规则。

## 1. 设计哲学

### 1.1 改装机设计的四条铁律

| 铁律 | 含义 | 违反后果 |
|---|---|---|
| **每台改装机有明确代价** | 被动越强，代价越大。没有免费午餐 | 改装机碾压，非改装机玩家无体验 |
| **特性必须可感知** | 玩家在 10 秒内能说出"这台改装机干了什么" | 认知过载，玩家不会选 |
| **反制必须存在** | 每台改装机都有破解方式，不存在无敌build | 环境固化，无策略深度 |
| **PvP 排位可选镜像** | 竞技模式中双方可选相同改装机，保证公平 | 养成差距碾压实竞技 |

### 1.2 机制分类框架

10 台改装机覆盖 6 种机制原型，确保玩法多样性：

| 机制原型 | 改装机 | 核心循环 |
|---|---|---|
| **状态累积** | 「霜降」CRYO、「炎核」PYRO | 反复用球打击对手 → 累积状态条 → 触发控制效果 |
| **连锁反应** | 「电涌」SURGE | 球碰挡板 → 触发电磁脉冲 → 影响其他球/玩家 |
| **场地操控** | 「锚点」ANCHOR、「工事」ENGINEER | 改变战场结构 → 影响所有球的路径 |
| **阻尼干扰** | 「阻尼」DAMPENER | 减速/锁定对手球 → 掌控节奏 |
| **反伤反击** | 「棘壁」BASTION、「蓄能」CAPACITOR | 被攻击 → 积累反击资源 → 爆发反攻 |
| **隐蔽突袭** | 「幽灵」GHOST | 隐藏球路 → 对手反应不及 → 得分 |
| **概率操控** | 「混沌」GLITCH | 随机穿墙/分裂 → 不可预测 → 制造混乱 |

### 1.3 状态累积系统（通用框架）

「霜降」和「炎核」共享一套**状态累积框架**，后续新改装机可复用：

```csharp
// 通用状态累积框架
public class StatusAccumulationSystem
{
    // 每名玩家维护一组状态条
    // Dictionary<PlayerId, Dictionary<StatusType, StatusState>>
}

public struct StatusState
{
    public StatusType Type;        // Cryo / Thermal / Charge / ...
    public float CurrentValue;     // 当前累积值 (0~MaxValue)
    public float MaxValue;         // 触发阈值 (默认100)
    public float DecayPerSecond;   // 自然衰减速率
    public float AccumulatePerHit; // 每次命中累积量
    public TriggerEffect Trigger;  // 达到阈值时触发的效果
    public float TriggerDuration;  // 触发效果持续时间
    public float PostTriggerReset; // 触发后重置值 (通常归零)
}

public enum StatusType
{
    None,
    Cryo,      // CRYO：冷冻锁定
    Thermal,   // PYRO：过热减速
    Charge,    // CAPACITOR：蓄能层数（自身累积，非对敌）
}

public enum TriggerEffect
{
    FreezePaddle,      // 锁定挡板（不可移动）
    SlowPaddle,        // 减速挡板
    SpeedBoostBall,    // 球速暴涨
    ReflectBall,       // 球反弹向敌方
}
```

**累积触发条件**：

| 触发点 | 说明 |
|---|---|
| `OnPaddleBounce` | 球碰挡板时，球归属者的改装机特性施加给挡板归属者 |
| `OnWallBounce` | 球碰墙时（部分改装机） |
| `OnGoalScored` | 进球时 |
| `OnGoalRebound` | 己方门回弹时 |

**衰减规则**：状态值持续衰减，鼓励持续进攻。如果对手拉开距离不打球，状态会慢慢消退。

## 2. 十改装机完整设计

### 2.1 「霜降」CRYO

| 属性 | 内容 |
|---|---|
| **代号** | `HERO_FROST_QUEEN`（内部 ID，不变） |
| **显示名** | 「霜降」CRYO |
| **主题色** | 冰蓝 #4FC3F7 |
| **元素** | 冷冻/低温 |
| **机制原型** | 状态累积 — Cryo |
| **玩家幻想** | "你的挡板每次碰我的球都会结霜，结满了就锁死。" |

#### 被动模块：冷冻涂层 (Cryo Coating)

「霜降」的球表面涂覆**冷冻液**。当对手用挡板反弹她的球时，对手挡板累积 **+15 冷冻值**。冷冻值达到 **100** 时，对手挡板被**锁定 1.5 秒**（完全无法移动），锁定结束后冷冻值归零。

| 参数 | 值 [PLACEHOLDER] | 说明 |
|---|---|---|
| 每次挡板反弹累积 | +15 | 约 7 次反弹触发锁定 |
| 触发阈值 | 100 | |
| 锁定时长 | 1.5s | 60s 局大约触发 2~3 次 |
| 冷冻衰减 | 5/s | 2s 不被碰就掉 10 点 |

**关键设计**：只有**挡板反弹**累积冷冻值，**墙壁反弹不累积**。这意味着对手可以选择"放球撞墙"来避免累积——但墙弹的角度不可控，球可能直接进门。

#### 主动模块：冷却过载 (Coolant Overload)

| 属性 | 值 |
|---|---|
| 效果 | 己方所有场上球的冷冻涂层浓度翻倍（+30/次），持续 5s |
| 冷却 | 25s |
| 初始冷却 | 8s（开局不能立刻用） |
| 视觉 | 冷冻液雾气扩散 + 己方球尾迹变浓白 |

**战术意图**：在已经累积了 60~70 冷冻值时开启，5 秒内 2 次挡板反弹即可锁定。是"收割"节奏的加速器。

#### 代价：低温损耗 (Cryo Drain)

| 代价 | 效果 |
|---|---|
| 球初速 -10% | 冷冻液增加球体重量，球飞得慢 |
| 球最大速度 -8% | 上限也低 |

**反制方式**：
1. 让她的球撞墙而非挡板——墙弹不累积冷冻值
2. 拉开节奏，利用冷冻衰减窗口
3. 选「阻尼」/「电涌」等远距离改装机，减少挡板接触

### 2.2 「炎核」PYRO

| 属性 | 内容 |
|---|---|
| **代号** | `HERO_PYRO_LORD`（内部 ID，不变） |
| **显示名** | 「炎核」PYRO |
| **主题色** | 烈焰橙 #FF6D00 |
| **元素** | 热能/过热 |
| **机制原型** | 状态累积 — Thermal |
| **玩家幻想** | "球上附着热能标记，挡板接多了就过热减速。" |

#### 被动模块：热能附着 (Thermal Tag)

「炎核」的球表面附着**热能标记**。对手用挡板反弹他的球时，累积 **+12 热量值**。热量达到 **100** 时，对手挡板**过热**：移动速度 **-50%**，持续 **2.5 秒**，过热结束后热量归零。

| 参数 | 值 [PLACEHOLDER] | 说明 |
|---|---|---|
| 每次挡板反弹累积 | +12 | 约 8~9 次反弹触发 |
| 触发阈值 | 100 | |
| 过热时长 | 2.5s | 比锁定长，但不是完全锁死 |
| 减速幅度 | -50% | 还能动，但很慢 |
| 热量衰减 | 4/s | 比冷冻慢，持续施压感更强 |

**与「霜降」的差异化**：
- 「霜降」= 完全锁定但时间短（1.5s），累积快（+15/次），衰减快（5/s）
- 「炎核」= 减速但时间长（2.5s），累积慢（+12/次），衰减慢（4/s）

「霜降」是"瞬间爆发控制"，「炎核」是"持续过热压迫"。

#### 主动模块：过载弹射 (Overcharge Shot)

| 属性 | 值 |
|---|---|
| 效果 | 下一次发球的球为**过载弹**：球速 +50%，碰撞半径 +50%，穿透 1 次墙体碰撞（不反弹直接穿过） |
| 冷却 | 20s |
| 初始冷却 | 6s |
| 视觉 | 过热光晕 + 橙色热能拖尾 |

**战术意图**：过载弹是"必进"级别的压迫工具。球速快、体积大、穿墙——对手挡板过热时发一颗，基本必进。但只有 1 颗，用完就没了。

#### 代价：热熔损耗 (Thermal Drain)

| 代价 | 效果 |
|---|---|
| 最大弹药 -1 | 热能核心占用更多能源，弹药容量更少 |

**反制方式**：
1. 热量衰减比冷冻慢，但过热只是减速不是锁定——可以顶着减速继续防守
2. 过载弹穿墙但不穿挡板——挡板仍然是最终防线
3. 「炎核」弹药少，打完弹药后有真空期

### 2.3 「电涌」SURGE

| 属性 | 内容 |
|---|---|
| **代号** | `HERO_THUNDER_ARBITER`（内部 ID，不变） |
| **显示名** | 「电涌」SURGE |
| **主题色** | 闪电黄 #FFD600 |
| **元素** | 电磁/电涌 |
| **机制原型** | 连锁反应 |
| **玩家幻想** | "球碰你挡板会释放电磁脉冲，偏转旁边的球路。" |

#### 被动模块：电磁偏转 (EM Deflect)

当「电涌」的球命中对手挡板时，触发**电磁脉冲**：从被命中的挡板向最近的 1 颗球（距离 ≤ 3 单位）释放一道电涌，该球获得一个**向最近敌方球门方向的微小偏转力**（速度不变，方向偏转 5°~8°）。

| 参数 | 值 [PLACEHOLDER] | 说明 |
|---|---|---|
| 电涌范围 | 3 单位 | |
| 偏转角度 | 5°~8° | 微小但可感知 |
| 脉冲目标 | 最近 1 颗球（不分归属） | 可能偏转己方球 |
| 冷却 | 2s（同一挡板触发后 2s 内不再脉冲） | 防止刷屏 |

**设计意图**：「电涌」不是直接控制对手，而是**扰动全场球路**。她的球打到你 → 旁边的球被电涌偏转了方向 → 可能飞向某个球门。制造"蝴蝶效应"。

#### 主动模块：全弹追踪 (Target Lock)

| 属性 | 值 |
|---|---|
| 效果 | 己方所有场上球获得**追踪协议**：球速 +50%，且每 0.5s 方向自动微调（单次最多偏转 10°）朝向最近敌方球门，持续 4s |
| 冷却 | 30s |
| 初始冷却 | 10s |
| 视觉 | 球体被电磁弧光包裹 + 球到敌方球门的淡金色引导线 |

**战术意图**：全弹齐射的追分窗口。所有己方球同时加速 + 自动追踪敌门——4 秒内制造密集得分压力。场上球越多效果越强，是"火力全开"的爆发节点。

**与被动的协同**：被动电磁脉冲在对手挡板附近制造球路偏转（扰动防守），主动让自己的球全速冲向敌门（强化进攻）。先用被动把对手节奏打乱，再开主动一波收割。

**边界说明**：
- 方向微调是渐进的（每 0.5s 最多 10°），不是瞬间转向——对手仍有反应窗口
- 微调只改方向不改速度大小，+50% 速度增益是独立的
- 球碰墙/挡板后微调方向重新计算（以当前位置到最近敌门为准）
- 场上无己方球时释放则无效（不消耗冷却）

#### 代价：轻量框架 (Light Frame)

| 代价 | 效果 |
|---|---|
| 挡板长度 -10% | 电磁设备占用空间，挡板更短 |

**反制方式**：
1. 球之间保持距离，避免电磁脉冲传导
2. 电磁脉冲可能偏转己方球——「电涌」自己也要小心球的位置
3. 挡板短，防守面积小，直接攻她的门

### 2.4 「锚点」ANCHOR

| 属性 | 内容 |
|---|---|
| **代号** | `HERO_GRAVITY_ENVOY`（内部 ID，不变） |
| **显示名** | 「锚点」ANCHOR |
| **主题色** | 深空紫 #7C4DFF |
| **元素** | 引力/锚定 |
| **机制原型** | 场地操控 |
| **玩家幻想** | "场地中心预装了我的引力锚，你的球路过就会被拉弯。" |

#### 被动模块：引力锚定 (Gravity Anchor)

场地中心始终存在一个**弱引力锚点**（影响半径 5 单位），所有**非己方**的球经过时受到向中心的微小加速度（0.3/s²）。「锚点」的己方球不受影响。

| 参数 | 值 [PLACEHOLDER] | 说明 |
|---|---|---|
| 引力半径 | 5 单位 | 约场地半径的 1/3 |
| 加速度 | 0.3/s² | 微小但可感知 |
| 影响对象 | 所有非己方球 | 己方球免疫 |

**设计意图**：对手的球飞过中心时会被微微"拉弯"。这改变了直线球路，让对手的瞄准变得困难——他们必须计算引力偏转。「锚点」自己的球不受影响，可以打直线。

#### 主动模块：引力塌缩 (Gravity Collapse)

| 属性 | 值 |
|---|---|
| 效果 | 在指定位置生成一个**高功率引力锚**（影响半径 6 单位），强烈吸引**所有球**（包括己方），持续 4s |
| 冷却 | 28s |
| 初始冷却 | 8s |
| 视觉 | 紫色引力网格 + 锚点标记 |

**战术意图**：引力塌缩是"全场的暂停键"——所有球被吸向一个点。可以用来：
- 阻止对手快攻（把球吸走）
- 聚集球后反弹出去打群攻
- 紧急防守时把飞向己方门的球吸偏

**关键**：引力塌缩也吸己方球，使用时机要精准。

#### 代价：发射延迟 (Launch Delay)

| 代价 | 效果 |
|---|---|
| 发球初速 -15% | 球要挣脱引力锚发射，初速更低 |

**反制方式**：
1. 被动引力锚只影响中心区域，走边路可以避开
2. 引力塌缩吸所有球，「锚点」自己也会被干扰
3. 发球初速低，她的球更容易被挡板拦截

### 2.5 「阻尼」DAMPENER

| 属性 | 内容 |
|---|---|
| **代号** | `HERO_TIME_WEAVER`（内部 ID，不变） |
| **显示名** | 「阻尼」DAMPENER |
| **主题色** | 时钟金 #BFA46F |
| **元素** | 阻尼/延迟 |
| **机制原型** | 阻尼干扰 |
| **玩家幻想** | "我的挡板周围有阻尼力场，你的球进来就减速。" |

#### 被动模块：阻尼力场 (Dampening Field)

以「阻尼」**挡板中心**为圆心，存在一个**减速力场**（半径 5 单位 [PLACEHOLDER]）。对手的球进入这个圆形区域时，球速 **-15%**；离开区域后恢复正常速度。减速力场**跟随挡板实时移动**。

| 参数 | 值 [PLACEHOLDER] | 说明 |
|---|---|---|
| 减速幅度 | -15% | 可感知但不致命 |
| 生效区域 | 挡板中心为圆心，半径 5 单位的圆 | 与「锚点」引力锚半径相同 |
| 区域跟随 | 实时跟随挡板中心移动 | 挡板移到哪，减速场就在哪 |
| 影响对象 | 所有非己方球 | 己方球不受影响 |
| 进出判定 | 每 tick 检查球与挡板中心距离 | 进入即减速，离开即恢复 |

**设计意图**：「阻尼」是防守型改装机。她的挡板是一个"阻尼力场"——对手球靠近就变慢，她有更多时间反应和防守。但减速场跟随挡板移动，这意味着：

1. **主动防守**：「阻尼」需要用挡板主动追踪来球方向，把球"罩"在减速场内。不是站着不动就能减速全场，而是需要操作。
2. **取舍决策**：移动挡板去减速一颗球，可能暴露另一个角度的防守空档。多球时需要判断优先罩哪颗。
3. **进攻不受限**：她的球出去打人时是正常速度——减速场只影响对手球，不影响己方。

#### 主动模块：系统冻结 (System Freeze)

| 属性 | 值 |
|---|---|
| 效果 | **锁定所有对手球** 1.5 秒（球原地停止），己方球继续移动 |
| 冷却 | 35s（长冷却，大招级） |
| 初始冷却 | 12s |
| 视觉 | 全场灰度滤镜 0.5s + 对手球上冻结时钟标记 |

**战术意图**：最强的节奏打断工具。对手球飞向你的门？冻结。你想进攻？对手球停了你打。1.5 秒在 60 秒局里是巨大的窗口。

**平衡**：35 秒冷却是所有改装机中最长的，一局最多用 1~2 次。

#### 代价：限速器 (Speed Limiter)

| 代价 | 效果 |
|---|---|
| 球最大速度 -10% | 阻尼力场溢出影响己方球速上限 |

**反制方式**：

2. 多球同时从不同角度进攻，「阻尼」只能罩住一个方向
3. 系统冻结只 1.5s，且冷却极长——不要被一次冻结打乱心态
4. 球速上限低，「阻尼」的进攻球更容易追上

**核心决策**：
- 进攻方："我要不要趁着她还瘦的时候疯狂得分？还是控节奏别让她太快长满？"
- 「棘壁」："我丢分了，但挡板长了——这值得。现在用加宽的挡板配合反弹装甲反击。"

3. 自生长护板让挡板越来越宽但越来越慢——前期快打远侧，后期瞄准远侧端点

2. 分裂球速低（×0.8），更好防守

4. 中性球进了谁的门都不计分——趁机用挡板碰它认领，而不是干等它漂走

## 4. 改装机与模块系统的集成

### 4.1 层级关系

```
改装机 (基底)
  定义: 被动模块 + 主动模块 + 代价
  生效: 整局持续
  
× 模块激活 (修饰)
  定义: 对 EffectiveMatchRule 的数值修饰
  生效: 激活后持续

= 最终对局规则
```

**关键原则**：改装机定义**机制**（"球带冷冻涂层"），模块修饰**数值**（"球速 +20%"）。模块不改变改装机的核心机制，只调节数值。

### 4.2 模块 × 改装机交互示例

| 模块 | 改装机 | 交互效果 |
|---|---|---|
| 蓄能击 (+反弹球速) | 「霜降」CRYO | 球更快，但对手挡板反弹次数可能变少（球太快不好接），冷冻累积变慢 |
| 蓄能击 (+反弹球速) | 「蓄能」CAPACITOR | 爆发 +100% 速 × 模块 +20% = 总 +120%，爆发球极难防守 |
| 长板 (+挡板长度) | 「幽灵」GHOST | 「幽灵」-15% + 模块 +25% = 净 +10%，代价被部分抵消 |
| 快装填 (+弹药回复) | 「炎核」PYRO | 弹药少(-1)但回复快，真空期缩短 |
| 分裂 (墙弹概率分裂) | 「混沌」GLITCH | 分裂球 + 穿墙 = 极度混乱 |
| 追踪弹 (球微调向敌门) | 「锚点」ANCHOR | 球被引力弯 + 模块微调 = 双重导向 |

### 4.3 交互规则

| 规则 | 说明 |
|---|---|
| 改装机代价 × 模块加成 | 模块加成可以部分抵消改装机代价，但不能完全消除（代价是改装机的底线） |
| 改装机被动 × 模块效果 | 独立生效，乘法叠加（如球速 = 基础 × 改装机修正 × 模块修正） |
| 改装机主动 × 过载事件 | 主动模块和过载事件独立计时，可同时生效 |
| 状态累积 × 模块 | 模块不改变累积速率（那是改装机机制），只改变球的基础属性 |

### 4.4 镜像模式下的改装机

PvP 排位使用镜像激活（模块），但**改装机仍然各自选择**：

| 模式 | 改装机 | 模块 |
|---|---|---|
| PVE | 各选 | 各自激活 |
| PVP 休闲 | 各选 | 各自激活 |
| PVP 排位 | **各选**（允许相同） | 镜像激活 |

改装机选择是玩家身份的表达，即使在排位中也允许不同改装机——改装机之间的平衡通过代价机制保证。

## 7. 平衡框架

### 7.1 改装机强度评估维度

| 维度 | 评估问题 | 权重 |
|---|---|---|
| 进攻压迫 | 被动+主动能制造多少得分机会？ | 高 |
| 防守稳固 | 被动+主动能阻止多少失分？ | 高 |
| 主动性 | 玩家操作空间有多大？ | 中 |
| 容错率 | 失误代价多大？ | 中 |
| 学习曲线 | 多久能上手？ | 低 |

### 7.2 改装机间平衡矩阵

每台改装机应该对至少 2 台改装机有优势，被至少 2 台改装机克制：

| 改装机 | 优势对 | 劣势对 | 理由 |
|---|---|---|---|
| 「霜降」CRYO | 「幽灵」、「混沌」 | 「阻尼」、「锚点」 | 锁定克制突袭型；「阻尼」减速球让冷冻球更慢 |
| 「炎核」PYRO | 「棘壁」、「蓄能」 | 「霜降」、「幽灵」 | 过热克制防守型；「霜降」同类型但累积更快 |
| 「电涌」SURGE | 「锚点」、「混沌」 | 「棘壁」、「幽灵」 | 连锁克制多球型；挡板短被突袭型打 |
| 「锚点」ANCHOR | 「蓄能」、「棘壁」 | 「电涌」、「幽灵」 | 引力克慢速防守型；发球慢被突袭 |
| 「阻尼」DAMPENER | 「霜降」、「炎核」 | 「幽灵」、「混沌」 | 减速克状态累积(球慢好挡)；突袭多角度绕开减速场 |
| 「棘壁」BASTION | 「幽灵」、「混沌」 | 「炎核」、「电涌」 | 反伤克突袭；弹药少被「炎核」反扣 |
| 「工事」ENGINEER | 「混沌」、「幽灵」 | 「锚点」、「阻尼」 | 屏障挡不可预测球；场地操控叠加混乱 |
| 「幽灵」GHOST | 「电涌」、「阻尼」 | 「霜降」、「棘壁」 | 迷彩克短挡板；锁定/反伤克制近身 |
| 「蓄能」CAPACITOR | 「炎核」、「电涌」 | 「混沌」、「幽灵」 | 蓄力克高频进攻；不可预测打乱蓄力 |
| 「混沌」GLITCH | 「阻尼」、「蓄能」 | 「工事」、「霜降」 | 混乱克规律型；屏障稳定球路 |

### 7.3 平衡红线

| 红线 | 值 | 理由 |
|---|---|---|
| 主动模块冷却下限 | ≥ 15s | 低于 15s 变成常规武器而非技能 |
| 主动模块冷却上限 | ≤ 40s | 高于 40s 一局用不了 2 次 |
| 主动模块效果时长上限 | ≤ 6s | 超过 6s 变成持续 buff 而非技能 |
| 状态累积触发时长上限 | ≤ 3s | 控制效果不能太长 |
| 单项代价上限 | ≤ -20% | 代价不能让改装机在基础玩法中残废 |
| 球数净增上限 | ≤ +5 | 分裂/多发不能让球数失控 |
| 锁定/完全控制时长上限 | ≤ 2s | 完全不能操作的时间不超过 2s |

## 8. 系统交互矩阵

| 系统A × 系统B | 交互规则 |
|---|---|
| 改装机被动 × 模块效果 | 乘法叠加，模块修数值，改装机改机制 |
| 改装机主动 × 能量波动 | 独立计时，主动模块不受波动影响 |
| 改装机主动 × 过载事件 | 可同时生效，效果乘法叠加 |
| 「霜降」冷冻 × 道具冻结 | 冻结效果不叠加（取较长者） |
| 「炎核」过载弹 × 模块穿透 | 穿透次数累加（过载弹穿1次 + 模块穿1次 = 穿2次） |
| 「混沌」分裂 × 能量波动 | 波动激活模块不影响已分裂的球 |
| 「蓄能」蓄能 × 模块反弹增强 | 反弹增强加成在蓄能爆发时额外乘算 |
| 「工事」屏障 × 地图墙体变量 | 屏障和地图墙体独立存在，均反弹球 |
| 「幽灵」迷彩 × 过载事件信号坍缩 | 坍缩瞬间球恢复可见，坍缩后恢复迷彩 |
| 「阻尼」系统冻结 × 能量波动 | 系统冻结期间波动仍触发（系统事件不受改装机影响） |
| 改装机代价 × 地图变量 | 代价作用于改装机基础属性，地图变量改变结构，独立计算 |
| 「霜降」冷冻 × 「炎核」热量 | 不同状态类型独立累积，互不干扰 |

## 12. 风险与缓解

### 12.1 确定性风险

| 风险 | 严重度 | 缓解 |
|---|---|---|
| 状态累积多端不同步 | 高 | 状态值每 tick 纳入 checksum |
| 穿墙协议伪随机不一致 | 高 | 使用 MatchSeed 派生的确定性 PRNG |
| 「工事」屏障位置不一致 | 高 | 屏障生成位置由 PRNG 种子决定 |
| 锁定/过热计时器多端不一致 | 中 | 计时器纳入运行时状态快照 |

### 12.2 设计风险

| 风险 | 严重度 | 缓解 |
|---|---|---|
| 状态累积改装机（冷冻/热能）过强 | 高 | 衰减机制 + 只有挡板反弹累积 + 代价 |
| 「混沌」的随机性破坏竞技体验 | 中 | 8% 失去归属作为自伤风险 |
| 「幽灵」迷彩让新手完全看不到球 | 中 | 拖尾仍可见 + 迷彩需 4 单位飞行距离 |
| 改装机间克制关系太硬导致选改装机就赢 | 中 | 克制是"优势"不是"必胜"，操作和模块仍重要 |
| 10 台改装机同时上线平衡工作量过大 | 高 | 分 2 批：先 3 个免费验证框架，再上 7 个 |

### 12.3 工程风险

| 风险 | 严重度 | 缓解 |
|---|---|---|
| BallRuntimeState 扩展字段影响序列化 | 中 | 新字段默认值兼容旧版本 |
| 改装机效果脚本化引入复杂度 | 中 | 用枚举+参数化实现，不用动态脚本 |
| 屏障/引力锚新增碰撞体影响 swept collision 性能 | 中 | 限制同时存在的额外碰撞体数量 |
| 改装机代价修改 EffectiveMatchRule 链路复杂 | 中 | 代价在 BuildEffectiveRule 时一次性应用，不中途变 |
