# Gatebreaker Arena 英雄芯片主线版本规划 v0.1

> **版本**: v0.1 | **日期**: 2026-07-08 | **状态**: 规划稿
> **目标**: 将 Gatebreaker Arena 的早期版本收敛为「基础弹球规则 + 英雄 + 芯片构筑」主线，降低局内随机、地图变量、道具事件等系统叠加带来的复杂度。

---

## 0. 规划结论

当前推荐先不同时推进「地图变量、局内道具、共鸣波动、量子事件、完整签名芯片商业化」。

早期版本应围绕一个更清晰的核心体验展开：

> 玩家选择英雄，配置芯片牌组，开局觉醒少量芯片，由英雄机制和芯片构筑改变本局弹球打法。

这样可以保留差异化、成长感和构筑策略，同时避免 60 秒短局内出现过多不可解释的随机变化。

---

## 1. 版本分层总览

| 版本 | 定位 | 核心目标 | 新增内容 | 暂不包含 |
|---|---|---|---|---|
| V0 基础验证版 | 核心弹球可玩 | 验证基础碰撞、得分、发球资源是否好玩 | 基础弹球规则、3 人/多人对局、计分、发球弹药 | 英雄、芯片、地图变量、道具 |
| V1 英雄构筑 MVP | 第一版主线 | 验证「英雄 + 芯片」是否成立 | 3 英雄、通用芯片 12 张、开局觉醒 3 张、2 条路径/英雄、M1/M2 | 签名芯片品级、M3、赛中追加觉醒 |
| V2 英雄路径扩展版 | 构筑深度 | 验证路径差异和长期养成 | 每英雄 3 路径、M3、签名芯片单品级、通用芯片 24 张 | 精铸/共鸣/至高品级、量子事件 |
| V3 养成与公平 PvP 版 | 可长期游玩 | 建立局外成长和排位公平框架 | 芯片升级、碎片经济、镜像/归一化排位、更多英雄 | 局内随机道具、复杂地图机关 |
| V4 完整商业化版 | 长线运营 | 提供英雄专属追求和付费空间 | 签名芯片品级、英雄礼包、赛季奖励、皮肤表现 | 仅在验证后选择性加入局内事件 |

## 2. V0 基础验证版

### 2.1 目标

确认没有任何英雄和芯片时，基础弹球对局本身是否稳定、有节奏、有可读性。

### 2.2 范围

| 模块 | 内容 |
|---|---|
| 基础规则 | 球运动、挡板反弹、球门得分 |
| 发球系统 | 弹药、冷却、最大弹药、发球方向 |
| 对局流程 | 倒计时、Playing、结算 |
| 平衡参数 | 球速、挡板长度、门宽、单局时长 |

### 2.3 验收标准

- 玩家能在 1 局内理解目标。
- 对局时长稳定在目标区间。
- 得分原因清楚，不出现大量无法解释的丢分。
- 碰撞、归属、发球资源在多人同步下稳定。

---

## 3. V1 英雄构筑 MVP

### 3.1 目标

验证「英雄差异 + 芯片构筑」是否能在不加入局内随机事件的情况下，让每局打法不同。

### 3.2 范围

| 模块 | 内容 |
|---|---|
| 英雄 | 3 个免费英雄 |
| 英雄路径 | 每个英雄 2 条路径 |
| 路径里程碑 | 仅 M1 / M2 |
| 通用芯片 | 12 张，攻/守/速/乱各 3 张 |
| 牌组 | 最多 8 张 |
| 觉醒 | 开局固定抽 3 张，整局不再追加 |
| 局外 | 简单芯片解锁，不做复杂经济 |

### 3.3 明确不做

- 不做 M3 极化。
- 不做签名芯片品级。
- 不做 t=20s / 40s / 55s 共鸣波动。
- 不做量子事件。
- 不做地图变量和局内道具。

### 3.4 推荐英雄数量

| 英雄 | 设计定位 |
|---|---|
| 冰雪女王 | 控速、防守、减速压制 |
| 烈焰领主 | 进攻、加速、持续压迫 |
| 荆棘守护者 | 防守反击、挡板成长、资源干扰 |

### 3.5 验收标准

- 玩家能说清自己选的英雄有什么差异。
- 玩家能在 30 秒内完成牌组选择。
- 一局结束后，玩家能复述本局觉醒的 3 张芯片。
- M1/M2 触发后能明显改变打法，但不会掩盖基础弹球操作。

---

## 4. V2 英雄路径扩展版

### 4.1 目标

在 V1 成立后，增加路径深度，让每个英雄拥有更明确的构筑方向。

### 4.2 范围

| 模块 | 内容 |
|---|---|
| 英雄路径 | 每个英雄扩展到 3 条路径 |
| 里程碑 | 加入 M3 极化 |
| 签名芯片 | 引入单品级签名芯片，每路径 1 张 |
| 通用芯片 | 扩展到 24 张 |
| 共鸣值 | 通用芯片 + 签名芯片共同计算路径共鸣值 |

### 4.3 明确不做

- 签名芯片暂时不分精铸/共鸣/至高。
- 签名芯片不做开箱和付费强化。
- 排位模式不允许签名芯片造成不可归一化的强度差。
- 仍不做赛中追加觉醒和量子事件。

### 4.4 验收标准

- 单路径深入、双路径均衡、三路径广撒都有可玩性。
- M3 是质变，但不是自动获胜。
- 签名芯片让英雄更有身份感，但不形成明显付费强度差。

---

## 5. V3 养成与公平 PvP 版

### 5.1 目标

建立可长期游玩的局外成长，并解决 PvP 公平性。

### 5.2 范围

| 模块 | 内容 |
|---|---|
| 碎片经济 | 胜负奖励、每日首胜、基础购买 |
| 芯片升级 | 通用芯片 Lv1/Lv2/Lv3 |
| 排位公平 | 镜像觉醒、等级归一化或官方租借满配 |
| 休闲 PvP | 可允许各自养成差异 |
| 新手保护 | 免费英雄 + 免费基础芯片 |

### 5.3 排位公平建议

排位模式必须满足至少一种：

| 方案 | 说明 |
|---|---|
| 等级归一化 | 所有芯片按统一等级生效 |
| 官方租借 | 双方临时使用同等配置池 |
| 完全镜像 | 双方觉醒完全相同的芯片效果 |

不建议在排位中直接使用「签名芯片各自觉醒 + 品级差异完整生效」。

### 5.4 验收标准

- 新手与老手在排位中的胜率差主要来自操作，而不是养成。
- 玩家能理解成长目标。
- 经济节奏不会在 1 周内耗尽，也不会形成明显付费墙。

---

## 6. V4 完整商业化版

### 6.1 目标

在核心玩法、英雄路径、PvP 公平和经济节奏都验证后，再引入完整签名芯片品级和商业化空间。

### 6.2 范围

| 模块 | 内容 |
|---|---|
| 签名芯片品级 | 精铸 / 共鸣 / 至高 |
| 专属表现 | 品级边框、特效、音效、英雄主题表现 |
| 获取渠道 | 指定购买、赛季奖励、英雄礼包 |
| 长期目标 | 每个英雄每条路径有可追求的最终形态 |

### 6.3 谨慎项

- 至高签名芯片不能在 PvP 排位中直接形成数值碾压。
- 品级差异优先表现为玩法风格、视觉身份和构筑效率，而不是纯面板强度。
- 如果引入随机箱，必须保留可指定购买路径。

---

## 7. 暂缓系统清单

以下系统不是取消，而是建议在 V3 之后再评估：

| 系统 | 暂缓原因 |
|---|---|
| 地图变量 | 会增加空间规则学习成本，影响英雄芯片平衡判断 |
| 局内道具 | 会干扰玩家对英雄和芯片强度的归因 |
| 共鸣波动 | 中途追加规则会增加信息负担和同步风险 |
| 量子事件 | 条件随机触发难以解释胜负原因 |
| 复杂动态机关 | 旋转墙、重力场、穿墙等会放大工程确定性风险 |

---

## 8. 推荐实施顺序

```
V0 基础弹球
  ↓
V1 3英雄 + 12通用芯片 + 开局3觉醒
  ↓
V2 3路径 + M3 + 单品级签名芯片
  ↓
V3 局外养成 + 公平PvP
  ↓
V4 签名品级 + 商业化 + 长线运营
  ↓
再评估地图变量 / 局内事件 / 量子事件
```

---

## 9. 核心原则

1. 先验证弹球手感，再验证英雄构筑。
2. 先做开局确定性构筑，再做赛中随机变化。
3. 先做少量英雄深度验证，再扩展到 10 英雄。
4. 排位公平优先于养成展示。
5. 玩家能解释输赢原因，比系统数量更多更重要。
