# 文档维护与任务收尾流程

这页固定 Gatebreaker Arena 的任务收尾和文档维护流程。

## Summary

- 代码完成不等于任务完成。
- 工程级改动完成后，应追加迭代记录并同步长期文档索引。
- 长期方案文档使用 `## 完成情况` 表达进度，不靠文件名表达状态。

## 常用命令

```bash
python3 tools/doc_maintenance/update_project_docs.py --doc-root doc sync
```

```bash
bash tools/doc_maintenance/finalize_task.sh \
  --summary "本轮完成了什么" \
  --done "已完成项" \
  --next "下一步" \
  --agent6-review not-required
```

## 方案完成情况格式

```text
## 完成情况

- 当前状态：未开始 / 进行中 / 已完成
- 进度说明：一句话说明当前整体完成到哪、剩余什么
- 最近更新：YYYY-MM-DD，本轮完成/确认的关键点
```

## 回复前建议

- 如果本轮改了工程文件，先执行文档同步。
- 如果本轮是明确里程碑，使用 `finalize_task.sh` 追加迭代记录。
- 如果只是纯问答或计划讨论，不强制写迭代记录。
