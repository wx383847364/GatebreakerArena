## 三、源代码文档格式要求

### 3.1 格式规范

- **页数**：前30页 + 后30页 = 60页（代码不足60页时全部提交）
- **每页行数**：50行
- **字号**：宋体小五号（10.5pt）或五号（10pt）
- **行距**：单倍行距
- **页眉**：标注软件名称和版本号
- **页码**：连续页码

### 3.2 内容要求

- 提交的代码应为本软件的核心代码，不包含自动生成代码
- 代码应可读，保留原始缩进和格式
- 代码中不应包含注释掉的废弃代码块
- 前后两部分代码应连续，不可跳跃选取

### 3.3 本项目源代码文档

本项目已使用 `tools/generate_copyright_source.py` 脚本自动生成源代码文档，位于：

```
doc/软著申请/源代码文档.md
```

该脚本自动完成以下工作：
1. 扫描 `Assets/` 目录下所有 `.cs` 文件
2. 按文件路径排序
3. 去除纯空行
4. 提取前1500行（前30页）和后1500行（后30页）
5. 按每页50行分页输出

### 3.4 转换为 Word 文档

由于版权保护中心要求提交 PDF 或 Word 格式，需将 Markdown 转换：

1. 使用 Typora、VS Code 等 Markdown 编辑器打开 `源代码文档.md`
2. 导出为 PDF 或 Word 格式
3. 添加页眉：`Gatebreaker Arena V1.0 源代码`
4. 确认每页50行代码，共60页

## 五、提交注意事项

1. **材料一致性**：申请表中的软件名称、版本号必须与设计说明书、源代码文档完全一致
2. **代码原创性**：源代码必须为开发者原创，不含第三方版权代码的核心逻辑
3. **完整性**：设计说明书应完整描述软件的功能、架构和技术特点
4. **格式规范**：所有材料应按版权保护中心要求格式提交
5. **申请人信息**：如以企业名义申请，需提供营业执照；以个人名义申请，需提供身份证
